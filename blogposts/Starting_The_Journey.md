# Starting the Journey
### 3/17/2025

This is the start of my blog.

I graduated with a Computer Science degree with a focus in Cybersecurity. I currently work as an analyst for an Application Security team, a role I got right after graduating after being converted from an intern to FTE. I haven't done a software engineering internship, and I haven't done many coding projects (or any at all) due to wanting to focus on Cybersecurity in college. I have grown to like the field of Application Security and aim to become an Application Security engineer. After doing some research online, I have found that a highly advantageous skill to have in the field is development experience. So, it's time to code something.

My goal with this coding project is to go through the entire Software Development Lifecycle and learn what it's like. In addition, I aim to embed Security within the lifecycle, going throgh the SSDLC. 

For the idea that I wanted to actually code up, I reflected on what inconveniences I have in my daily life that I might be able to alleviate with software. One such idea is making outfits. I have a very basic sense of fashion, so I was thinking of coding something that can keep track of my wardrobe and help create outfits. I thought of the following base features the application should have. 

## FitFinder Phase 1: Offline Outfit Builder
### Base Features
- [ ] Upload articles of clothing that are then saved in a wardrobe
- [ ] Tag attributes of the clothing uploaded
- [ ] Be able to build an outfit by selecting different garment types (shirts, pants, jackets, accessories)

I felt like these two features made up a pretty good Minimum Viable Product, but obviously my brainstorming didn't stop there. I wanted an application that helped me with my fundamental problem, my basic sense of style. So, I thought up some features that would help me design outfits.

## FitFinder Phase 2: Fashion Assistant
- [ ] Calculate which colors complement a piece of clothing
- [ ] Determine which pieces of clothing go with a selected piece of clothing based on tags

I knew an app like this probably already existed, so I looked online to see if what I had in my head already existed, and I found a site that was pretty close to what I was planning in my head. It had some features I wanted to incorporate, but I also wanted to add features that it didn't have to create a product with more functionality.

## FitFinder Phase 3: Pushing the Boundaries
### Existing Features
- [ ] Utilizing AI to create outfits
- [ ] Choosing styles to create outfits
- [ ] Specifying weather/season to create outfits
- [ ] Creating outfits based on a selected article of clothing

### Additional Missing Features
- [ ] More styles (lookastic only has relaxed, casual, smart casual, and dressy)
    - Examples: Soft boy, goth, techwear, etc.
    - [ ] New styles accompanied by wikis that describe the style and the general practices to dress in that style
- [ ] Outfit calendar that allows you to plan outfits in advance per day and pulls weather in your area for that day
- [ ] Clothing auto-tagging

And the only thing about the UI I had in mind is that the Wardrobe UI should looks like a closet with cd jukebox like scrolling. 

I think I'll leave the brainstorming there, moving onto the first step of determining requirements should be pretty straightforward since I've laid out the features in a requirements-like format.
